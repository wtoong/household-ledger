# 보안 / 프라이버시

## 위협 모델 한 줄 요약
완전 클라이언트(정적 웹) 앱입니다. 백엔드·서버·로그인이 없고, 거래 데이터는 **사용자 브라우저의 IndexedDB에만** 저장됩니다.

## 저장소를 public으로 공개하면 무엇이 노출되나?
- 공개되는 것: **소스 코드뿐**입니다.
- 공개되지 **않는** 것: 사용자의 **금융 거래 데이터**. 이 데이터는 깃 저장소에 들어 있지 않고, 각자 브라우저의 IndexedDB(앱을 연 출처별 로컬 저장소)에만 있습니다. 저장소를 public으로 바꿔도 데이터는 노출되지 않습니다.
- 단, 앞으로 절대 **실제 거래 파일(CSV/XLSX)이나 내보낸 JSON을 커밋하지 마세요.** (`.gitignore`에 데이터 확장자를 추가하는 것을 권장)

## 적용된 보호 장치
- **외부 전송 차단**: 코드에 `fetch`/`XMLHttpRequest`/WebSocket/외부 CDN 호출이 전혀 없습니다. CSP `connect-src 'none'`로 네트워크 사용 자체를 막아 데이터 유출 경로를 차단합니다.
- **CSP**: `script-src 'self'`(인라인 스크립트 금지), `object-src 'none'`, `base-uri 'none'`, `form-action 'none'`. `<meta http-equiv>`로 적용.
- **추적기 없음**: 분석/광고 스크립트 미포함.
- **XSS 방어**: 사용자 입력(적요/가맹점/날짜/소스)을 DOM에 넣을 때 모두 이스케이프 처리.
- **referrer 차단**: `referrer: no-referrer`.

## 서드파티 의존성
- **SheetJS (xlsx) 0.18.5** — XLSX 파싱용으로 `vendor/`에 동봉(오프라인). `eval`/`Function` 미사용이라 엄격 CSP와 호환됩니다.
- 참고: 0.19.3 미만에는 프로토타입 오염(CVE-2023-30533) 이슈가 있습니다. 본 앱은 (a) 사용자 본인 파일만 파싱하고 (b) `sheet_to_json({header:1})` 배열 모드로 읽어 객체 키 주입 표면이 작아 실사용 위험은 낮습니다. 네트워크가 허용되면 공식 배포처(cdn.sheetjs.com)의 **0.20.2+**로 교체를 권장합니다.

## GitHub Pages 호스팅 시 유의
- `<username>.github.io/<repo>` 형태로 호스팅하면 같은 계정의 다른 Pages 프로젝트와 **출처(origin)를 공유**합니다. 같은 출처의 다른 페이지가 이 앱의 IndexedDB에 접근할 수 있으므로, 민감 데이터 격리가 중요하면 전용 계정이나 커스텀 도메인 사용을 고려하세요.
